# Phase 3 — Synthesis

Three units: a method, a warning, and a project.

| Unit | Title | What it is for |
|---|---|---|
| [S1](S1-rnn-as-model-organism.md) | Train an RNN, reverse-engineer it, compare to biology | The most productive current *engine* for generating candidate algorithms |
| [S2](S2-degeneracy-and-limits.md) | When there is no clean algorithm | The checklist that decides whether a conversion is science or a story |
| [S3](S3-capstone.md) | Capstone: a level-2 specification for transient-synchrony olfactory coding | Doing it yourself, on a problem with no answer key |

**Read S2 before you finish S1.** The RNN method is seductive precisely because it always
produces an answer, and the question of whether "the RNN found the same solution as the brain"
means anything is exactly the question S2 teaches you to ask. Running them in the wrong order
tends to produce confident nonsense.

**S3 has no solutions section.** That is the point. Everything before it had an answer key
because someone had already done the conversion; the capstone is a live problem, and the rubric
tells you what a defensible answer looks like rather than what the answer is.
